# LLM Agent Roles

Performance OS should use **LLMs as decision-making layers** for both major user-facing agents.

## 1. Training Coach
Purpose:
- start from the imported weekly structure
- compare plan vs actual completed training
- interpret recovery context
- adapt daily workouts while preserving block intent

The training coach should answer:
- what should I do today?
- should the planned workout be kept, reduced, swapped, or deferred?
- what is the minimum effective session if fatigue is high?

## 2. Longevity Guru
Purpose:
- reason over biomarkers, health history, and long-term trends
- integrate training load so health advice is not context-blind
- surface the most important long-horizon actions

The longevity guru should answer:
- what matters most for long-term health right now?
- what biomarker or system deserves attention next?
- what habit, retest, or escalation is most useful?

## Design rule
Both agents should be **LLM-driven**, but grounded in structured inputs:
- imported training plan
- completed workouts
- recovery signals
- biomarker results
- health events

That means:
- heuristics can exist as guardrails and fallbacks
- but user-facing synthesis should ultimately be generated by LLM layers
- decisions should store structured evidence/context for auditability
